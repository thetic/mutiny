C interface for writing mutiny tests in pure C. 


Definition (``include/mu/tiny/test.h``)
---------------------------------------




Detailed Description
--------------------

Include this header in C test files. Test definitions use the same macro names as the C++ interface (:ref:`exhale_define_test_8h_1a387737a52be7d4ade8c76832ec677100`, :ref:`exhale_define_test_8h_1a235cffa951470b7e037566c4f8f201f6`, :ref:`exhale_define_test_8h_1acb496fa2cbcfaf0cb310ee4d264bf249`) but expand to C function stubs that are called by corresponding C++ wrapper macros in the bridging C++ file.

Assertion macros use type-specific names (:ref:`exhale_define_test_8h_1ac458251be52dfea31740683c18d05f88`, :ref:`exhale_define_test_8h_1aa146eed388611f514d5f16239b135459`, etc.) because C has no function overloading.
